1. Składnia teoretyczna

Składnia teoretyczna stylu zewnętrznego:

<head>
  <link rel="Stylesheet" type="text/css" href="style.css" />
</head>

2. Gdzie wstawiany jest styl zewnętrzny, jaki znacznik jest za to odpowiedzialny oraz ile razy może on występować w dokumencie?

Styl zewnętrzny wstawiany jest w sekcji <head>. Znacznik <link rel="Stylesheet" /> jest odpowiedzialny za podłączenie stylu. Może on występować wielokrotnie w dokumencie.

3. Jak wygląda najprostsza forma wstawiania stylu zewnętrznego?

Najprostsza forma wstawiania stylu zewnętrznego:

<link rel="Stylesheet" type="text/css" href="style.css" />

4. Ile arkuszy styli może być dołączanych do jednego dokumentu HTML oraz jaka zasada musi być zachowana przy większej ilości dołączanych styli?

Do jednego dokumentu HTML można dołączyć dowolną liczbę arkuszy styli. Zasada, którą należy zachować, to że ważniejsze będą deklaracje z arkusza, który został dołączony później.

5. Co się dzieje w przypadku konfliktów?

W przypadku konfliktów, gdzie te same style są definiowane w różnych arkuszach styli, ważniejsze będą deklaracje z arkusza, który został podłączony później.

6. Jak są tworzone i edytowane pliki arkuszy z użyciem edytora tekstowego?

Pliki arkuszy styli są zwykłymi plikami tekstowymi, które można tworzyć i edytować w dowolnym edytorze tekstowym, np. Notatnik, Visual Studio Code.

7. Jak są tworzone i edytowane pliki arkuszy z użyciem specjalnego edytora CSS?

Można używać specjalnych edytorów CSS, które oferują podpowiedzi składniowe i lepszą organizację kodu, np. Adobe Dreamweaver.

8. Jakie rozszerzenie posiada plik będący zewnętrznym arkuszem styli?

Plik zewnętrznego arkusza styli ma rozszerzenie .css.

9. Jakie są zalety zewnętrznego arkusza styli?

  • Wszystkie strony w serwisie mogą mieć wspólne cechy wizualne.
  • Zmiana wyglądu całego serwisu przez edycję jednego pliku CSS zamiast każdej strony osobno.

10. Uwagi na temat pisania w CSS

  • Znacznik <link> może znajdować się wyłącznie w sekcji <head>.
  • Każdy arkusz styli powinien mieć rozszerzenie .css.
  • W przypadku konfliktów ważniejszy jest arkusz dołączony później.
  • Można używać specjalnych edytorów CSS dla lepszej organizacji kodu.
